概觀

 

Microsoft .Net 1.1 提供的 API 可以用來裝載不含 IIS 的 Aspx 應用程式。.Net 2.0 Beta1 版本則提供建置於 Http.Sys 上的 HttpListener 類別，讓使用者能夠建立獨立的 Http 伺服器。本範例示範如何結合 HttpListener 的功能來建立 Http 伺服器，然後再由 Http 伺服器將呼叫傳送到裝載的 Aspx 應用程式。

 

建置範例

您必須執行下列步驟，才能正確建置及執行本範例。

使用 Visual Studio 2005 Beta 開啟專案檔 AspxHostCS.sln

接著，按一下 [建置] -> [建置專案] 來建置專案。

執行範例

若要執行本範例，請遵循下列其中一項指示。

  • 從 Visual Studio 2005 Beta 執行專案
  • 開啟命令提示字元，然後移至範例所在的位置。從該位置切換至 bin\Debug 目錄。執行 AspxHostCS.exe 應用程式。

執行範例之前，請先參閱注意 -> 範例用法章節。

注意

使用的 HttpListener 功能

 

  1. 驗證
  2. 啟用 SSL
  3. 讀取安全連接的用戶端憑證

 

類別描述

 

AspxHostCS.cs e 建立及設定接聽程式和 Aspx 應用程式的主要類別。

AspxVirtualRoot.cs e 設定 HttpListener 接聽前置詞與支援之驗證配置的類別。

AspxNetEngine.cs e 透過指派對應至實體目錄的虛擬別名來設定 Aspx 應用程式的類別。

AspxPage.cs e 實作 SimpleWorkerRequest 類別並代表用戶端所要求頁面的類別。

AspxRequestInfo.cs e 可用來將相關資料從 HttpListenerContext 傳送到裝載應用程式的資料保存類別。

AspxException.cs e 自訂例外狀況類別。

DemoPages e 包含範例 Aspx 頁面的目錄。

 

範例用法

 

AspxHostCS.cs 檔案是一種類別，其包含的主要方法會啟動 HttpListener，並將實體目錄設定成裝載的 Aspx 應用程式。這個類別預設會嘗試將 DemoPages 目錄 (位於同一個範例目錄) 設定為虛擬別名 / 下的裝載應用程式。如果沒有變更，按一下提供的連結將會顯示這個讀我檔案頁面。由於本範例中的 HttpListener 會接聽通訊埠 80，因此您可能需要停止 IIS 才能執行本範例。

 

請修改程式碼，以進行適合個別用途的變更。

 

//如有必要，建立具有 http 通訊埠和 https 通訊埠的 AspxVirtualRoot 物件

AspxVirtualRoot virtualRoot = new AspxVirtualRoot(80);

 

//將實體目錄設定為虛擬別名。

//TODO: 將實體目錄取代成要設定的目錄。

virtualRoot.Configure("/", Path.GetFullPath(@"..\..\DemoPages"));

 

//TODO: 如果要加入驗證，請在這裡加入

//virtualRoot.AuthenticationSchemes = AuthenticationSchemes.Basic;

 

設定驗證配置

 

設定 AspxVirtualRoot 物件之後，請設定 AspxVirtualRoot 物件上的 AuthenticationScheme 欄位，以設定必要的驗證配置。

 

 

啟用 SSL

 

若要啟用 SSL，您必須在需要 SSL 的通訊埠上，設定安裝在電腦存放區的伺服器憑證。如需如何使用 Httpcfg.exe 公用程式在通訊埠上設定伺服器憑證的詳細資訊，請參閱 Httpcfg 連結。

 

注意：您也可以使用 Winhttpcertcfg 在通訊埠上設定伺服器憑證，請參閱 Winhttpcertcfg 連結。

FAQ

 

啟動應用程式時收到下列錯誤訊息

 

"System.IO.FileNotFoundException: 無法載入檔案或組件 'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null' 或其相依性的其中之一。系統找不到指定的檔案。檔案名稱：'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null'"

 

解決方法：

 

AspxHostCs.exe 檔案不在目前正在設定之實體目錄的 bin 目錄中。請將 AspxHostcs.exe 檔案複製到 bin 目錄中。
