概述

 

随 Microsoft .Net 1.1 一起提供的 API 可用于承载 Aspx 应用程序，而无需 IIS。用户使用随 .Net 2.0 的 beta1 版一起提供的、在 Http.Sys 上生成的 HttpListener 类可以创建独立的 Http 服务器。本演示说明如何对 HttpListener 的功能进行组合以创建 Http 服务器，该服务器将传送对承载的 Aspx 应用程序的调用。

 

生成示例

要正确生成并运行本示例，需要执行下列步骤。

使用 Visual Studio 2005 测试版打开项目文件 AspxHostCS.sln

然后，单击“生成”->“生成 项目”以生成项目。

运行示例

要运行本示例，请按照下列说明之一执行操作。

  • 在 Visual Studio 2005 测试版中运行项目
  • 打开命令提示符，转至示例所在位置。从该位置切换到 bin\Debug 目录。执行 AspxHostCS.exe 应用程序。

运行示例前，请参阅说明->示例用法一节。

说明

使用的 HttpListener 功能

 

  1. 身份验证
  2. 启用 Ssl
  3. 通过安全连接读取客户端证书

 

类说明

 

AspxHostCS.cs è 用于创建和配置侦听器和 Aspx 应用程序的主类。

AspxVirtualRoot.cs è 用于配置 HttpListener 以侦听前缀和所支持的身份验证方案的类。

AspxNetEngine.cs è 通过分配映射到物理目录的虚拟别名来配置 Aspx 应用程序的类。

AspxPage.cs è 实现 SimpleWorkerRequest 类并表示客户端请求的页的类。

AspxRequestInfo.cs è 用于将相关数据从 HttpListenerContext 传递到承载的应用程序的数据容器类。

AspxException.cs è 自定义异常类。

DemoPages è 包含示例 Aspx 页的目录。

 

示例用法

 

AspxHostCS.cs 文件是包含主方法的类，该主方法将启动 HttpListener 并将物理目录配置为承载的 Aspx 应用程序。默认情况下，该类会将 DemoPages 目录（在同一个示例目录中）配置为虚拟别名 / 下某个承载的应用程序。如果未做更改，则单击给出的链接将提供此自述文件页。 由于本示例中的 HttpListener 侦听端口 80，因此，可能需要停止 IIS 才能运行本示例。

 

更改代码以适合各种用途。

 

               //如果需要，创建具有 http 端口和 https 端口的 AspxVirtualRoot 对象

               AspxVirtualRoot virtualRoot = new AspxVirtualRoot(80);

 

               //将物理目录配置为虚拟别名。

               //TODO:使用要配置的目录来替换物理目录。

               virtualRoot.Configure("/", Path.GetFullPath(@"..\..\DemoPages"));

 

               //TODO:如果要添加身份验证，请在此处添加

               //virtualRoot.AuthenticationSchemes = AuthenticationSchemes.Basic;

 

设置身份验证方案

 

配置 AspxVirtualRoot 对象之后，在 AspxVirtualRoot 对象中设置 AuthenticationScheme 字段，以设置所需的身份验证方案。

 

 

启用 Ssl

 

要启用 Ssl，必须在要求 Ssl 的端口上配置服务器证书，该证书安装在计算机存储区中。有关如何使用 Httpcfg.exe 实用工具在端口上配置服务器证书的更多信息，请参阅 Httpcfg 链接。

 

注意：也可以使用 Winhttpcertcfg 在端口上配置服务器证书，请参阅 Winhttpcertcfg 链接。

常见问题

 

启动应用程序时，出现以下错误信息

 

 “System.IO.FileNotFoundException:未能加载文件或程序集‘AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null’或它的某一个依赖项。系统找不到指定文件。文件名:'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null'”

 

解决方法:

 

AspxHostCs.exe 文件不在所配置的物理目录的 bin 目录中。将 AspxHostcs.exe 文件复制到 bin 目录中。
