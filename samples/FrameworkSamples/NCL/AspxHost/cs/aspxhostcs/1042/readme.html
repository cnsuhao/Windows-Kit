개요

 

Microsoft .Net 1.1에는 IIS를 사용하지 않고 Aspx 응용 프로그램을 호스팅하는 데 사용할 수 있는 API가 제공되었습니다. .Net 2.0 베타1 버전에는 독립 실행형 Http 서버를 만드는 데 사용할 수 있는 Http.Sys에 기초한 HttpListener 클래스가 제공되었습니다. 이 데모에서는 HttpListener의 기능을 조합하여 호스팅된 Aspx 응용 프로그램에 대한 호출을 차례로 라우팅하는 Http 서버를 만드는 방법을 보여 줍니다.

 

샘플 빌드

이 샘플을 제대로 빌드하여 실행하려면 다음 단계를 수행해야 합니다.

Visual Studio 2005 베타에서 AspxHostCS.sln 프로젝트 파일을 엽니다.

그런 다음 [빌드] > [프로젝트  빌드]를 클릭하여 프로젝트를 빌드합니다.

샘플 실행

이 샘플을 실행하려면 아래 나열된 지침 중 하나를 따르십시오.

  • Visual Studio 2005 베타에서 프로젝트를 실행합니다.
  • 명령 프롬프트를 열고 샘플이 있는 위치로 이동합니다. 해당 위치에서 bin\Debug 디렉터리로 변경합니다. AspxHostCS.exe 응용 프로그램을 실행합니다.

샘플을 실행하기 전에 참고 > 샘플 사용 단원을 참조하십시오.

클래스 설명

 

AspxHostCS.cs è 수신기와 Aspx 응용 프로그램을 만들고 구성하는 주 클래스입니다.

AspxVirtualRoot.cs è 지원되는 접두사와 인증 체계를 수신 대기하기 위해 HttpListener를 구성하는 클래스입니다.

AspxNetEngine.cs è 실제 디렉터리에 매핑하는 가상 별칭을 지정하여 Aspx 응용 프로그램을 구성하는 클래스입니다.

AspxPage.cs è SimpleWorkerRequest 클래스를 구현하고 클라이언트에서 요청하는 페이지를 나타내는 클래스입니다.

AspxRequestInfo.cs è HttpListenerContext에서 관련 데이터를 호스팅된 응용 프로그램으로 전달하는 데 사용되는 데이터 소유자 클래스입니다.

AspxException.cs è 사용자 지정 예외 클래스입니다.

DemoPages è샘플 Aspx 페이지가 들어 있는 디렉터리입니다.

 

샘플 사용

 

AspxHostCS.cs 파일은 HttpListener를 시작하고 실제 디렉터리를 호스팅된 Aspx 응용 프로그램으로 구성하는 main 메서드가 들어 있는 클래스입니다. 기본적으로 이 클래스는 DemoPages 디렉터리(동일한 샘플 디렉터리 안에 있음)를 가상 별칭 / 아래에 호스팅된 응용 프로그램으로 구성합니다. 변경하지 않는 경우 제공된 링크를 클릭하면 이 추가 정보 페이지가 표시됩니다.  이 샘플의 HttpListener는 포트 80에서 수신하므로 이 샘플을 실행하려면 IIS를 중지해야 할 수도 있습니다.

 

변경 내용이 각각의 사용에 적합하도록 코드를 수정합니다.

 

                //필요한 경우 http 포트 및 https 포트를 사용하여 AspxVirtualRoot 개체를 만듭니다.

                AspxVirtualRoot virtualRoot = new AspxVirtualRoot(80);

 

                //실제 디렉터리를 가상 별칭으로 구성합니다.

                //TODO: 실제 디렉터리를 구성할 디렉터리로 바꿉니다.

                virtualRoot.Configure("/", Path.GetFullPath(@"..\..\DemoPages"));

 

                //TODO: 인증을 추가하려면 여기에 추가합니다.

                //virtualRoot.AuthenticationSchemes = AuthenticationSchemes.Basic;

 

인증 체계 설정

 

AspxVirtualRoot 개체를 구성한 후 AspxVirtualRoot 개체에서 AuthenticationScheme 필드를 설정하여 필요한 인증 체계를 설정합니다.

 

 

SSL 사용

 

SSL을 사용하려면 컴퓨터 저장소에 설치되어 있는 서버 인증서가 SSL을 사용할 포트에 구성되어 있어야 합니다. Httpcfg.exe 유틸리티를 사용하여 포트에서 서버 인증서를 구성하는 방법에 대한 자세한 내용은 Httpcfg 링크를 참조하십시오.

 

참고: Winhttpcertcfg도 포트에서 서버 인증서를 구성하는 데 사용할 수 있습니다. Winhttpcertcfg 링크를 참조하십시오.

FAQ

 

응용 프로그램을 시작할 때 다음과 같은 오류 메시지가 표시됩니다.

 

 “System.IO.FileNotFoundException: 파일이나 어셈블리 'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null' 또는 여기에 종속되어 있는 파일이나 어셈블리 중 하나를 찾을 수 없습니다. 지정한 파일을 찾을 수 없습니다. 파일 이름: 'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null'”

 

해결 방법:

 

구성할 실제 디렉터리의 bin 디렉터리에 AspxHostCs.exe 파일이 없습니다. AspxHostcs.exe 파일을 bin 디렉터리에 복사하십시오.
