Vue d'ensemble

 

Microsoft .Net 1.1 était fourni avec une API qui pouvait être utilisée pour héberger des applications Aspx sans IIS. Puis, la version bêta1 du .Net 2.0 était fournie avec la classe HttpListener construite sur Http.Sys qui permettait aux utilisateurs de créer un serveur Http autonome. Cette démonstration montre comment combiner les fonctionnalités de HttpListener pour créer un serveur Http qui route les appels vers l'application Aspx hébergée.

 

Génération de l'exemple

Afin de générer et d'exécuter correctement cet exemple, procédez comme suit.

Ouvrez le fichier projet AspxHostCS.sln dans Visual Studio 2005 Bêta

Générez le projet en cliquant sur Générer-> Générer un projet.

Exécution de l'exemple

Afin d'exécuter cet exemple, suivez les instructions indiquées ci-dessous.

  • Exécutez le projet dans Visual Studio 2005 Bêta
  • Ouvrez une invite de commandes et accédez à l'emplacement de cet exemple. Ouvrez le dossier bin\Debug à partir de cet emplacement. Exécutez l'application AspxHostCS.exe.

Avant d'exécuter cet exemple, reportez-vous à la section Remarques->Utilisation de l'exemple.

Description de la classe

 

AspxHostCS.cs è Classe principale qui crée et configure un écouteur et une application Aspx.

AspxVirtualRoot.cs è Classe qui configure un HttpListener pour écouter les préfixes et les schémas d'authentification pris en charge.

AspxNetEngine.cs è Classe qui configure une application Aspx en assignant un alias virtuel qui est mappé à un répertoire physique.

AspxPage.cs è Classe qui implémente la classe SimpleWorkerRequest et représente une page demandée par le client.

AspxRequestInfo.cs è Classe conteneur de données qui est utilisée pour passer les données appropriées de HttpListenerContext à l'application hébergée.

AspxException.cs è Classe d'exception personnalisée.

DemoPages èRépertoire qui contient les pages d'exemple Aspx

 

Utilisation de l'exemple

 

Le fichier AspxHostCS.cs est la classe qui contient la méthode principale qui lancera un HttpListener et configurera un répertoire physique sous la forme d'une application ASPX hébergée. Par défaut, la classe essaie de configurer le répertoire DemoPages (qui se trouve dans le même répertoire samples) sous la forme d'une application hébergée sous alias virtuel /. S'il n'est pas modifié, cette page Readme s'affichera si vous cliquez sur le lien fourni.  Dans la mesure où HttpListener dans cet exemple écoute le port 80, il se peut que vous deviez arrêter IIS pour exécuter cet exemple.

 

Modifiez le code afin qu'il réponde à vos besoins.

 

                //Crée un objet AspxVirtualRoot avec un port http et un port https si nécessaire

                AspxVirtualRoot virtualRoot = new AspxVirtualRoot(80);

 

                //Configure un répertoire physique sous la forme d'un alias virtuel.

                //TODO : remplacez le répertoire physique par le répertoire à configurer.

                virtualRoot.Configure("/", Path.GetFullPath(@"..\..\DemoPages"));

 

                //TODO : si l'authentification doit être ajoutée, ajoutez-la ici

                //virtualRoot.AuthenticationSchemes = AuthenticationSchemes.Basic;

 

Définition du schéma d'authentification

 

Après avoir configuré un objet AspxVirtualRoot, définissez le schéma d'authentification requis en définissant le champ AuthenticationScheme de l'objet AspxVirtualRoot.

 

 

Activation de Ssl

 

Pour activer Ssl, un certificat de serveur installé dans le magasin de l'ordinateur doit être configuré sur le port qui requiert Ssl. Pour plus d'informations sur la configuration d'un certificat de serveur sur un port à l'aide de l'utilitaire Httpcfg.exe, reportez-vous au lien Httpcfg.

 

Remarque : Winhttpcertcfg peut également être utilisé pour la configuration d'un certificat de serveur sur un port, reportez-vous au lien Winhttpcertcfg.

FAQ

 

Lorsque je démarre l'application, le message d'erreur suivant s'affiche

 

 “System.IO.FileNotFoundException: Impossible de charger le fichier ou l'assembly 'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null' ou une de ses dépendances. Le système ne trouve pas le fichier spécifié.Nom de fichier : 'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null'”

 

Solution :

 

Le fichier AspxHostCs.exe n'est pas présent dans le répertoire bin du répertoire physique en cours de configuration. Copiez le fichier AspxHostcs.exe dans le répertoire bin.
