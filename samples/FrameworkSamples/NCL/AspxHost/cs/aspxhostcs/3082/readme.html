Información general

 

Las API incluidas en Microsoft .Net 1.1 que se pueden utilizar para alojar las aplicaciones Aspx sin IIS. Después la versión beta1 incluida en la clase HttpListener de .Net 2.0 generada sobre Http.Sys que permitió a los usuarios crear un servidor Http independiente. Esta demo muestra el modo de combinar las características de HttpListener para crear un servidor Http que a cambio enrute las llamadas a la aplicación Aspx alojada.

 

Generar el ejemplo

Es necesario realizar los siguientes pasos para que este ejemplo se genere y se ejecute correctamente.

Abra el archivo del proyecto AspxHostCS.sln utilizando la versión beta de Visual Studio 2005

A continuación, genere el proyecto haciendo clic en Generar-> Generar proyecto.

Ejecutar el ejemplo

Para ejecutar este ejemplo siga una de las instrucciones siguientes.

  • Ejecute el proyecto desde la versión beta de Visual Studio 2005
  • Abra un símbolo del sistema y vaya a la ubicación del ejemplo. Cambie al directorio bin\Debug desde esa ubicación. Ejecute la aplicación AspxHostCS.exe.

Antes de ejecutar el ejemplo, consulte la sección Notas->Uso del ejemplo.

Notas

Características de HttpListener utilizadas

 

  1. Autenticación
  2. Habilitar Ssl
  3. Leer certificados de cliente sobre conexiones seguras

 

Descripción de la clase

 

AspxHostCS.cs è La clase principal que crea y configura un agente de escucha y una aplicación Aspx.

AspxVirtualRoot.cs è La clase que configura una clase HttpListener para escuchar en prefijos y esquemas de autenticación compatibles.

AspxNetEngine.cs è La clase que configura una aplicación Aspx asignándole un alias virtual que se asigna a un directorio físico.

AspxPage.cs è La clase que implementa la clase SimpleWorkerRequest y que representa una página solicitada por el cliente.

AspxRequestInfo.cs è La clase del contenedor de datos que se utiliza para pasar datos importantes desde HttpListenerContext a la aplicación alojada.

AspxException.cs è La clase de excepción personalizada.

DemoPages èEl directorio que contiene las páginas del ejemplo Aspx

 

Uso del ejemplo

 

El archivo AspxHostCS.cs es la clase que contiene el método principal que iniciará una clase HttpListener y que configura un directorio físico como una aplicación Aspx alojada. De manera predeterminada, la clase intenta configurar el directorio DemoPages (que se encuentra en el mismo directorio de ejemplos) como una aplicación alojada bajo un alias virtual /. Si no cambia, al hacer clic en el vínculo proporcionado se mostrará la página de este archivo Léame. Como la clase HttpListener de este ejemplo escucha en el puerto 80, es posible que tenga que detener IIS para ejecutar este ejemplo.

 

Cambie el código para efectuar los cambios adecuados al uso individual.

 

//Crear un objeto AspxVirtualRoot con un puerto http y un puerto https si fuese necesario

AspxVirtualRoot virtualRoot = new AspxVirtualRoot(80);

 

//Configurar un directorio físico como alias virtual.

//TODO: Sustituir el directorio físico por el directorio que se va a configurar.

virtualRoot.Configure("/", Path.GetFullPath(@"..\..\DemoPages"));

 

//TODO: Si es necesario agregar autenticación, agréguela aquí

//virtualRoot.AuthenticationSchemes = AuthenticationSchemes.Basic;

 

Establecer el esquema de autenticación

 

Después de configurar un objeto AspxVirtualRoot, establezca el esquema de autenticación necesario estableciendo el campo AuthenticationScheme en el objeto AspxVirtualRoot.

 

 

Habilitar Ssl

 

Para habilitar Ssl, un certificado de servidor instalado en el almacén de la máquina debe estar configurado en el puerto en el que se requiere Ssl. Para obtener más información sobre el modo de configurar certificados de servidor en un puerto mediante la utilidad Httpcfg.exe, consulte el vínculo Httpcfg.

 

Nota: Winhttpcertcfg también se puede utilizar para configurar certificados de servidor en un puerto. Consulte el vínculo Winhttpcertcfg.
